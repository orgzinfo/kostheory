The Quantification Problem

Organizational factors such as "culture," "leadership," and "strategic alignment" are inherently qualitative. They resist direct measurement. The Data-Theoretic Neurosymbolic Architecture (DTNA v3.0) solves this through four mechanisms operating in sequence:

  1. Structural decomposition — Each qualitative concept is decomposed into atomic BPM cells (node × position × domain × construct = 6,615 cells), where each cell asks a specific, observable question.
  2. Literature-derived priors (DT-BASE) — Academic evidence establishes Bayesian priors for each cell before any company is assessed.
  3. Site-specific evidence updating (DT-SITE) — Company operational data updates those priors via Beta-conjugate Bayesian inference.
  4. Position-sensitive polarity (PSM) — The same evidence type receives different signed weights depending on the PDCA position, converting qualitative observations into signed quantitative scores.

DT-BASE: Literature-Derived Priors

DT-BASE is the baseline literature corpus module. It is computed once, before any company is assessed. For each of the 6,615 BPM cells, K evidence entries are collected from the academic corpus via 19,845 cell-level research queries. Each entry j provides three metrics:

MetricNameMethodScale
M₁Source CredibilityAutomated: w₁·JIF + w₂·citations + w₃·h-index + w₄·recency[0, 1]
M₂Causal StrengthSemi-automated: LLM initial estimate, human validation (±0.1 for ~80% of entries)[0, 1]
M₃Analyst ConfidenceHuman only: set per domain per pillar, constant within analyst's expertise[0, 1]

The per-entry normalized weight and aggregated conditional probability (Valentino 2019):

wj = (M₁j × M₃j) / ∑(M₁k × M₃k)

p̂n,s,d = ∏(M₂jwj)     (weighted geometric mean)

Prior hyperparameters (κ = concentration, target κ ∈ [5, 10]):

α₀ = p̂n,s,d · κ      β₀ = (1 − p̂n,s,d) · κ

π₀(pn,s,d) ~ Beta(α₀, β₀)

DT-SITE: Company Evidence Updating

DT-SITE ingests the actual organization's data — HRIS, financial systems, Slack, documents, session transcripts — and updates the DT-BASE priors. Documents of type t at position s are weighted by the document-type credibility scale. Evidence counts:

E⁺n,s,d = ∑t wt,s · c⁺t,n,s,d     (positive evidence)

E⁻n,s,d = ∑t wt,s · c⁻t,n,s,d     (negative/absent evidence)

Bayesian Integration: DT-BASE + DT-SITE

The Beta-conjugate posterior for each cell at a specific site:

π(pn,s,d,c,site | E) ~ Beta(α₀ + E⁺, β₀ + E⁻)

Point estimate (cell quality score):

p̄ = (α₀ + E⁺) / (α₀ + E⁺ + β₀ + E⁻)

The Virtue field V enters as the concentration parameter: high V produces tighter priors (the system trusts the founder's evidence more); low V produces flatter priors (the system relies more on the literature baseline).

Role of S-Expressions

KOS uses Lisp S-expressions as its specification language. The same specification compiles to all production deployment targets. S-expressions serve four functions:

Homoiconicity
In Lisp, code and data share the same tree structure. A defnode declaration simultaneously specifies what a node contains and defines how to evaluate it. The framework definition is the evaluation program.
Recursive Self-Similarity
The PDCA pattern repeats at pillar, node, and position levels. A single Lisp macro generates all 64 node evaluation programs from one template because the structure is self-similar at every scale.
Multi-Formalism Compilation
One Lisp spec compiles to Firestore document schemas, BigQuery table definitions, Neo4j graph structures, Cloud Run services, and API contracts. One specification, multiple deployment targets.
Auditability
Every evaluation path is traceable through the S-expression tree. The symbolic spec can be read, reviewed, and formally verified by a human. Required for a system that produces organizational recommendations.
